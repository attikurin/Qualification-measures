生成AIパスポート試験とは

分野別攻略法と絶対暗記ポイント

第1章：AI基礎

AI・機械学習・深層学習の基本概念

AI（人工知能）の定義

人間の知的な振る舞いを模倣・支援するコンピュータシステム

  • 1956年のダートマス会議で「人工知能（Artificial Intelligence）」という用語が初めて使用
  • ソフトウェア技術（頭脳）
  • 判断・推論・学習などの知的処理

ロボットとの違い

物理的な動作を自動で行う機械装置

  • ハードウェア（身体）
  • センサーやアクチュエータを持つ
  • AIとロボットは別物

AIの種類

弱いAI（ANI）

特定のタスクに特化したAI

  • 現在実用化されているAIはすべて弱いAI
  • 例：画像認識、音声認識、チェスAI

強いAI（AGI）

人間と同等の汎用的な知能を持つAI

  • まだ実現していない
  • 様々なタスクを人間のように実行
絶対暗記： 1956年ダートマス会議 / AI = ソフトウェア / 現在のAI = すべて弱いAI（ANI）

教師あり学習

正解ラベル付きデータで学習

  • 画像分類（猫/犬の判別）
  • 価格予測
  • スパムメール検出

教師なし学習

ラベルなしデータからパターンを発見

  • クラスタリング（顧客グループ分け）
  • 次元削減（データ圧縮）
  • 異常検知

強化学習

試行錯誤で最適な行動を学習

  • 報酬を最大化する方針を獲得
  • ゲームAI（囲碁・将棋）
  • ロボット制御、自動運転

半教師あり学習

少量のラベル付き+大量のラベルなしデータ

  • ラベル付けコストの削減
  • 両方のデータを組み合わせて学習
  • 実用的な手法
絶対暗記： 教師あり = ラベル付き / 教師なし = ラベルなし / 強化 = 報酬 / 半教師あり = 混合

ニューラルネットワーク

人間の脳の神経細胞（ニューロン）の仕組みを模倣した機械学習モデル

  • 入力層・隠れ層・出力層の3層構造
  • ノード（ニューロン）とエッジ（重み）で構成

深層学習（ディープラーニング）

隠れ層を多層にしたニューラルネットワーク

  • 画像認識・音声認識で高性能
  • 大量のデータと計算資源が必要
  • CNN（画像）、RNN（時系列）、Transformer（言語）
絶対暗記： 機械学習 ⊇ ニューラルネットワーク ⊇ 深層学習

第2章：生成AI

ChatGPTや画像生成AIなど、コンテンツを生成するAI技術

CNN（畳み込みニューラルネットワーク）

画像認識に特化

  • 画像の特徴を自動抽出
  • 畳み込み層とプーリング層

VAE（変分オートエンコーダ）

データの圧縮と生成

  • 潜在空間を学習
  • 新しいデータの生成

GAN（敵対的生成ネットワーク）

リアルな画像生成

  • 生成器と識別器の対立
  • 高品質な画像生成

RNN・LSTM

時系列データ処理

  • 前の情報を記憶
  • テキスト・音声処理

Transformer

革新的なアーキテクチャ

  • Attention機構（注意機構）
  • 並列処理が可能
  • GPTやBERTの基盤技術
絶対暗記： CNN（画像）→ GAN（画像生成）→ RNN/LSTM（時系列）→ Transformer（革命）

GPTモデルの進化

  • GPT-1：Transformerベースの言語モデル
  • GPT-2：15億パラメータ、テキスト生成能力向上
  • GPT-3：1750億パラメータ、Few-shot学習
  • GPT-3.5：ChatGPTの初期版
  • GPT-4：マルチモーダル対応（テキスト+画像）
  • GPT-4o：オムニモーダル（テキスト+画像+音声）
  • GPT-o1、o3、o4：推論能力特化モデル

ChatGPT関連技術

  • Operator：ブラウザ操作可能なAI
  • Codex：コード生成特化モデル
  • DALL-E：Image Generation（画像生成）
  • RLHF：人間のフィードバックからの強化学習
  • ハルシネーション：AIの幻覚（誤った情報生成）
絶対暗記： GPT-4o = オムニモーダル / RLHF = 人間フィードバック学習 / ハルシネーション = 幻覚

LLM（大規模言語モデル）

テキスト生成AI

  • ChatGPT（OpenAI）
  • Claude（Anthropic）
  • Gemini（Google）
  • Copilot（Microsoft）

画像生成AI

テキストから画像を生成

  • DALL-E、Midjourney
  • Stable Diffusion
  • 拡散モデル（Diffusion Model）
  • データ拡張、リマスタリング

動画・音声生成AI

マルチメディア生成

  • 動画：Sora、Runway Gen-3、Veo3
  • 音声：TTS（Text-to-Speech）
  • 音楽：Suno、Udio

RAG（Retrieval-Augmented Generation）

外部データベースから情報を検索して、LLMの回答に活用する技術

  • 最新情報や企業固有の情報を参照可能
  • ハルシネーション（幻覚）の低減
  • モデル自体は変更しない

ファインチューニング

特定のタスクやドメインに合わせてモデルを追加学習させる

  • 医療・法律など専門分野への対応
  • 企業独自のスタイルに適応
  • モデルのパラメータを調整
絶対暗記： RAG = 外部情報の参照 / ファインチューニング = モデルの追加学習

トークン

LLMが処理するテキストの最小単位

  • 英語：約4文字で1トークン
  • 日本語：約1.5文字で1トークン
  • API利用料金はトークン数で計算

コンテキストウィンドウ

LLMが一度に処理できるトークンの最大数

  • GPT-4：128,000トークン（約19万文字）
  • Claude 3：200,000トークン
  • 長い文書も一度に処理可能

第3章：AI動向

最新のAI技術トレンドと将来展望

AIエージェントとは

自律的にタスクを実行し、ツールを使い分けながら目標を達成するAIシステム

  • 複数のツール・APIを自動で選択
  • 計画立案と実行を繰り返し
  • 例：Auto-GPT、BabyAGI、LangChain Agents

MCP（Model Context Protocol）

AIと外部システムの連携プロトコル

  • 統一的な方法で外部ツール・データにアクセス
  • APIやデータベースとの接続を標準化
  • AIエージェントの機能拡張を実現

主要AIエージェントツール

  • GenSpark：自動検索・分析エージェント
  • Manus：タスク自動化エージェント
  • Skywork AI：マルチタスク対応エージェント
絶対暗記： AIエージェント = 自律行動 / MCP = 外部連携プロトコル

マルチモーダルAIとは

テキスト・画像・音声・動画など複数のモダリティ（様式）を同時に処理できるAI

  • GPT-4V（Vision）：画像理解可能
  • Gemini：動画・音声・テキストを統合処理
  • 画像を見て説明文を生成

エッジAI

  • デバイス上で処理（スマホ、IoT機器）
  • 低遅延、プライバシー保護
  • オフラインでも動作
  • モデルは小型・軽量

クラウドAI

  • サーバー上で処理
  • 大規模モデルの利用可能
  • 高性能だが通信遅延あり
  • データをクラウドに送信

第4章：リテラシー・法律

最重要章：AI新法、個人情報保護、著作権など

主な脅威

フィッシング攻撃

  • フィッシング詐欺：偽サイトで個人情報を盗む
  • スミッシング：SMSを使ったフィッシング
  • ヴィッシング：音声通話を使ったフィッシング

マルウェア

  • マルウェア：悪意のあるソフトウェア全般
  • ランサムウェア：データを暗号化して身代金要求
  • トロイの木馬、スパイウェア

ソーシャルエンジニアリング

  • 人間の心理的な隙を突く攻撃
  • なりすまし、肩越し覗き見
  • 技術ではなく人を狙う
絶対暗記： フィッシング = 偽サイト / ランサムウェア = 身代金 / ソーシャルエンジニアリング = 人を狙う

AI社会の基本理念（3つのD）

  • Dignity（尊厳）：人間の尊厳を尊重
  • Diversity & Inclusion（多様性と包摂性）：多様な人々を包摂
  • Sustainability（持続可能性）：環境・社会の持続可能性

AI社会原則（7つの原則）

  1. 人間中心の原則：人間の幸福を最優先
  2. 安全性の原則：AIの安全性を確保
  3. 公平性の原則：差別やバイアスを排除
  4. プライバシー保護の原則：個人情報を保護
  5. セキュリティ確保の原則：セキュリティリスクに対応
  6. 透明性の原則：AIの動作を説明可能に
  7. アカウンタビリティの原則：説明責任を果たす

AI事業者ガイドライン（2024年4月公表）

日本政府が定めたAI開発・利用の指針

  • AI開発者：安全で公平なAIを開発
  • AI提供者：適切な情報提供と管理
  • AI利用者：責任ある利用

AI新法（2025年6月交付）

AIの開発・利用に関する法的枠組み

  • AI事業者ガイドラインを法制化
  • 高リスクAIへの規制強化
  • 透明性・説明責任の義務化
絶対暗記： 3つのD（Dignity、Diversity、Sustainability） / 7つの原則（人間中心、安全性、公平性、プライバシー、セキュリティ、透明性、アカウンタビリティ）

要配慮個人情報

特に慎重な取扱いが必要な個人情報

  • 人種、信条、社会的身分
  • 病歴、健康診断結果
  • 犯罪歴、犯罪被害
  • 本人の同意なく取得・利用禁止

AIと個人情報

  • 学習データに個人情報を含む場合は注意
  • 生成結果に個人情報が含まれる可能性
  • 匿名加工情報の活用
絶対暗記： 要配慮個人情報 = 人種・病歴・犯罪歴など（本人同意が必須）

知的財産権の種類

  • 著作権：創作物の保護（死後70年）
  • 特許権：発明の保護（20年）
  • 商標権：ブランドの保護（10年、更新可）
  • 意匠権：デザインの保護（最長25年）
  • 肖像権：人物の顔・姿の権利
  • パブリシティ権：著名人の経済的価値

AI学習と著作権

  • 著作権法第30条の4：AI学習目的での著作物利用は原則OK
  • ただし、海賊版や違法コンテンツは対象外
  • 著作権者の利益を不当に害する場合は違法

AI生成物の著作権

  • 原則：AI生成物そのものに著作権なし
  • 人間の創作的関与があれば著作権発生
  • プロンプトだけでは創作性不足

不正競争防止法

  • 営業秘密：秘密管理された有用な技術・営業情報
  • 限定提供データ：特定の者に提供されるデータ
  • 不正取得・使用・開示を禁止
絶対暗記： 著作権法第30条の4 = AI学習OK / AI生成物 = 原則著作権なし / 営業秘密 = 不正競争防止法

ディープフェイク

AIを使って本物そっくりの偽画像・動画を生成する技術

  • フェイクニュースの拡散
  • なりすまし詐欺
  • 肖像権・名誉毀損の問題
  • 選挙妨害のリスク

AIの倫理原則

  • 透明性と説明可能性（Explainability）
  • 公平性とバイアス排除
  • プライバシー保護
  • アカウンタビリティ（説明責任）

第5章：プロンプトエンジニアリング

効果的なプロンプトの作成技法

LM（言語モデル）とLLM

  • LM（Language Model）：言語の確率分布を学習するモデル
  • n-gramモデル：n個の単語の連続から次の単語を予測
  • LLM（Large Language Model）：大規模な言語モデル
  • プレトレーニング：大量のテキストデータで事前学習

ハイパーパラメータ（超重要）

Temperature（温度）

出力のランダム性を制御

  • 低い値（0.0～0.3）：確定的で一貫性のある出力
  • 高い値（0.7～1.0）：創造的で多様な出力

Top-p（核サンプリング）

候補単語の選択範囲を制御

  • 累積確率がp以下の単語から選択
  • 0.9～0.95が一般的
絶対暗記： Temperature低 = 確定的 / Temperature高 = 創造的 / Top-p = 候補範囲

プロンプトの4つの構成要素

  1. Instruction（指示）：AIに何をしてほしいか明確に指示
  2. Context（文脈）：背景情報や状況の説明
  3. Input Data（入力データ）：処理対象のデータ
  4. Output Indicator（出力形式）：期待する出力の形式

効果的なプロンプトの書き方

  • 具体的に指示：曖昧な表現を避ける
  • 役割を与える：「あなたはマーケティング専門家です」
  • 形式を指定：表形式、箇条書き、Markdown など
  • 例を示す（Few-shot）：期待する出力の例を提示
  • 制約条件を明示：文字数、トーン、対象読者など
絶対暗記： プロンプト4要素 = Instruction、Context、Input Data、Output Indicator

Zero-shot Prompting

例を示さず、指示だけで回答させる

例：「次の文を英訳してください」

Few-shot Prompting

数個の例を示してから質問

例：「ポジティブ:嬉しい、ネガティブ:悲しい → 次の文は？」

Chain-of-Thought（思考の連鎖）

「ステップバイステップで考えて」と指示

推論や計算問題で効果的

絶対暗記： Zero-shot = 例なし / Few-shot = 例あり / Chain-of-Thought = 段階的思考

プロンプトインジェクションとは

悪意のあるプロンプトを注入して、AIに意図しない動作をさせる攻撃

  • システムプロンプトの無視や上書き
  • 機密情報の漏洩
  • 有害コンテンツの生成

対策

  • 入力の検証とサニタイゼーション
  • システムプロンプトとユーザー入力の分離
  • 出力のフィルタリング
